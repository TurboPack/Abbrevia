TAbZipKit.PasswordRetries

TAbZipKit

property PasswordRetries: Byte

Default: 3

Specifies the maximum number of passwords to try when attempting to extract an encrypted file.

When the number of retries is exhausted, an exception is raised. See OnProcessItemFailure event in the printed documenation for a description of how exceptions that occur during the extract process are handled.

If PasswordRetries is 0 and Password is empty, encrypted files cannot be extracted.

See also: TAbZipBrowser.Password, OnNeedPassword, PasswordRetries
